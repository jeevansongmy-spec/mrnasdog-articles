## Sell pressure: where new VIRTUAL comes from

Sell #1 — protocol inflation — is **zero**. The full **1 billion VIRTUAL** was minted at launch against a fixed maximum supply, and Virtuals Protocol issues no new tokens; there is no staking emission, no block reward, no inflation schedule. With no mint authority producing fresh coins, the largest sell-side line on most chains simply does not exist here.

Sell #2 — vesting unlocks — is also **zero**: every original public-sale, ecosystem and liquidity allocation finished vesting back in 2023, so VIRTUAL is fully unlocked and no cliff lands inside this window. Sell #3 — Foundation and unscheduled unlocks — is zero as a flow; the one team-controlled holding worth naming is the **~343M Ecosystem Treasury** (the 35% allocation), but it carries no published release schedule and no dated release in the window, so the framework books it at zero and monitors it. Sell #4 — long-term locked or bankruptcy — is zero, because no bankruptcy estate or court distribution applies to VIRTUAL.

## Buy pressure: where new VIRTUAL goes

Buy #1 — programmatic buyback — is **zero for VIRTUAL**, and this is the subtle part of the story. Virtuals Protocol does run a revenue-funded buyback-and-burn, but it buys back and burns each **agent's own token** — names like GAME, CONVO and AIXBT — using VIRTUAL accumulated from post-bonding trading revenue across a time-weighted average. The VIRTUAL it spends flows back into the open market; the tokens destroyed are the agent tokens, not VIRTUAL. So the program reduces agent-token supply, not VIRTUAL supply, and it earns no buy-side credit on the VIRTUAL ledger.

Buy #2 — protocol fee burn — is zero: there is no base-fee mechanism that destroys VIRTUAL. Buy #3 — Foundation buy — and Buy #4 — new long-term lock — are both zero, with no discretionary open-market buying of VIRTUAL by the team and no new escrow or multi-year lock announced in the window. Every buy-side line that would remove VIRTUAL from the float reads zero.

## Foundation and overhang

VIRTUAL has exactly one overhang worth tracking: the **~343M Ecosystem Treasury**, the 35% of supply reserved for ecosystem growth, grants and liquidity. It is not a vesting cliff on a calendar — it is a discretionary pool the team can deploy, which is why the framework watches its balance rather than projecting a forward number. There is no observed firing inside the trailing window and no published schedule, so it stays at zero. The buyback program needs no accumulation wallet here, since bought-back value is spent on agent tokens, not parked. The standing rule holds: if the treasury balance falls between refreshes, that outflow enters Sell #3 at the next walk.

## How VIRTUAL compares to other fixed-supply tokens

VIRTUAL belongs to the class of **hard-capped, fully-distributed utility tokens** — closer to a fixed-supply asset than to an emission chain. Unlike an uncapped proof-of-stake L1 that mints validator rewards every block, VIRTUAL has no issuance at all; unlike a halving-model coin that still emits on a decaying curve, VIRTUAL is already fully minted and unlocked, so there is no forward emission to model. The supply side is simply settled.

What separates VIRTUAL from an exchange token that buys back and burns its own supply is the destination of the burn. Many revenue-sharing tokens route fees into buying and destroying the token itself, which is genuinely deflationary. Virtuals Protocol instead directs its revenue into agent-token burns, so the deflation accrues to the agents, not to VIRTUAL. For an inflation lens, that places VIRTUAL in a narrow group: a token whose supply is neither inflated nor deflated, where value would have to come from demand for the routing currency rather than from a shrinking float.
